# WinTrain 模块边界

## iOS 客户端

### 负责

- 录制视频、选择动作、上传文件
- 展示首页、结果页、历史页、我的页、订阅页
- 保存 `install_id` 到 Keychain
- 缓存配额快照，供 UI 展示
- 保存本地历史记录和本地派生片段
- 发起 StoreKit 2 购买与恢复购买

### 不负责

- 直接调用 LLM
- 决定是否扣减免费次数
- 持久化完整原始视频到历史
- 维护订阅权威状态

## Go 后端

### 负责

- 接收视频上传
- 文件大小、格式、动作类型校验
- 关键帧抽取
- LLM 调用与结构化解析
- `status` 三态标准化
- 服务端最终配额裁决
- 订阅状态查询与恢复绑定
- 操作日志记录

### 不负责

- 返回视频二进制给客户端
- 提供历史记录查询 API
- 持久化原始视频
- iOS 端 UI 逻辑

## LLM Provider

### 负责

- 接收关键帧序列和 Prompt
- 输出结构化 JSON 文本

### 不负责

- 配额判断
- 历史记录
- 订阅状态
- 用户设备识别

## 本地历史 vs 后端日志

用户可见历史与后端日志分离：

- iOS 本地历史：面向用户，用于“下次复看”
- 后端操作日志：面向诊断和运营，不直接暴露给用户

仅 `status=success` 的分析结果写入本地历史。
