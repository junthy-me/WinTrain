# WinTrain 开发工作流

1. 产品或设计变更必须先从 OpenSpec 开始。
2. Claude Code 用于：
   - 项目审计
   - 架构 / 设计
   - 任务拆分
   - 代码 Review
3. Codex 用于：
   - 有边界的实现任务
4. 任何实现过程中发现的设计冲突，都必须先回到 OpenSpec 更新 Artifact
5. 每个任务都必须定义：
   - 目标 (goal)
   - 输入
   - 非目标 (non-goal)
   - 修改文件范围
   - 验证方式
   - 验收标准
6. 每个合并的 change 都必须通过：
   - 后端单元测试
   - 客户端编译检查
   - 人工冒烟测试
